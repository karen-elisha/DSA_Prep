# HashMap in Java ‚Äî Complete DSA Notes

## 1. What is a HashMap?

`HashMap<K, V>` stores data as **key-value pairs**. It implements the `Map` interface and is backed by an **array of buckets**, where each bucket holds a linked list (or, since Java 8, a balanced tree for large buckets).

```java
import java.util.HashMap;

HashMap<String, Integer> map = new HashMap<>();
map.put("apple", 1);
map.put("banana", 2);
```

**Key properties:**
- No duplicate keys (adding an existing key overwrites the value)
- Allows **one null key** and **multiple null values**
- **Unordered** ‚Äî no guarantee of insertion order or sorted order
- Not thread-safe (use `ConcurrentHashMap` for thread safety)

---

## 2. Internal Working (Important for Interviews)

1. **Hashing**: `hashCode()` of the key is computed, then Java applies a supplemental hash function to spread bits and reduce collisions:
   ```java
   static final int hash(Object key) {
       int h;
       return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
   }
   ```
2. **Bucket index**: `index = hash & (capacity - 1)` (capacity is always a power of 2, so this is equivalent to `hash % capacity` but faster).
3. **Collision handling**:
   - Multiple keys landing in the same bucket are stored as a **linked list**.
   - Since **Java 8**: if a bucket's linked list grows beyond **8 nodes** (`TREEIFY_THRESHOLD`) AND table capacity ‚â• 64, it's converted to a **Red-Black Tree** ‚Üí worst case lookup becomes `O(log n)` instead of `O(n)`.
4. **Equality check**: When keys collide, `equals()` is used to check if it's truly the same key or just a hash collision.

**This is why overriding both `hashCode()` and `equals()` correctly matters** for custom objects used as keys.

---

## 3. Load Factor & Resizing

- **Default initial capacity**: 16
- **Default load factor**: 0.75
- **Threshold** = capacity √ó load factor (e.g., 16 √ó 0.75 = 12)
- When `size > threshold`, the map **resizes (doubles capacity)** and **rehashes all entries** ‚Äî an expensive `O(n)` operation.

```java
HashMap<String, Integer> map = new HashMap<>(32, 0.5f); // custom capacity & load factor
```

**Tip**: If you know the approx. number of entries beforehand, initialize with adequate capacity to avoid repeated resizing:
```java
new HashMap<>(expectedSize * 4 / 3 + 1);
```

## 4. Time Complexity

| Operation | Average Case | Worst Case (pre-Java 8 / bad hash) |
|-----------|--------------|--------------------------------------|
| `put()`   | O(1)         | O(n) ‚Üí O(log n) with treeification  |
| `get()`   | O(1)         | O(n) ‚Üí O(log n) with treeification  |
| `remove()`| O(1)         | O(n) ‚Üí O(log n)                     |
| `containsKey()` | O(1)   | O(n)                                 |
| `containsValue()` | O(n) | O(n) ‚Äî always linear (scans all values) |

## 5. Commonly Used Methods

```java
map.put(key, value);            // insert or update
map.get(key);                   // returns value or null
map.getOrDefault(key, default); // returns default if key absent
map.remove(key);                // removes entry
map.containsKey(key);           // boolean
map.containsValue(value);       // boolean, O(n)
map.size();
map.isEmpty();
map.clear();

map.keySet();       // Set<K> view of keys
map.values();        // Collection<V> view of values
map.entrySet();      // Set<Map.Entry<K,V>> ‚Äî best for iteration

// Java 8+ functional methods (very useful in DSA)
map.putIfAbsent(key, value);
map.merge(key, value, (oldV, newV) -> oldV + newV);   // great for frequency maps
map.compute(key, (k, v) -> v == null ? 1 : v + 1);
map.computeIfAbsent(key, k -> new ArrayList<>());     // great for adjacency lists
map.computeIfPresent(key, (k, v) -> v + 1);
map.forEach((k, v) -> System.out.println(k + "=" + v));
```

## 6. Iterating a HashMap

```java
// Best practice ‚Äî entrySet (single pass, gets key & value together)
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " -> " + entry.getValue());
}

// keySet (needs a lookup for value ‚Äî slightly less efficient)
for (String key : map.keySet()) {
    System.out.println(key + " -> " + map.get(key));
}

// values only
for (int val : map.values()) {
    System.out.println(val);
}
```

## 7. Custom Objects as Keys

If you use a custom class as a key, **override both**:

```java
class Point {
    int x, y;
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Point)) return false;
        Point p = (Point) o;
        return x == p.x && y == p.y;
    }

    @Override
    public int hashCode() {
        return Objects.hash(x, y);
    }
}
```
**Rule**: Equal objects (by `equals()`) **must** have the same `hashCode()`. Violating this breaks HashMap lookups silently (you'll insert but never retrieve).

## 8. HashMap vs Other Map Implementations

| Feature | HashMap | LinkedHashMap | TreeMap |
|---|---|---|---|
| Order | No order | Insertion order | Sorted by key |
| Null keys | 1 allowed | 1 allowed | Not allowed |
| Underlying structure | Array + LinkedList/Tree | Array + doubly linked list | Red-Black Tree |
| get/put | O(1) avg | O(1) avg | O(log n) |
| Use case | Fast lookups | Order matters (e.g. LRU cache) | Sorted keys needed |

**HashSet** internally uses a HashMap where values are a dummy constant object ‚Äî so `HashSet` = `HashMap` with only keys mattering.

## 9. Classic DSA Patterns Using HashMap

### a) Frequency Counting
```java
Map<Character, Integer> freq = new HashMap<>();
for (char c : s.toCharArray()) {
    freq.merge(c, 1, Integer::sum);
}
```

### b) Two Sum (O(n) using HashMap)
```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[]{map.get(complement), i};
        }
        map.put(nums[i], i);
    }
    return new int[]{-1, -1};
}
```

### c) Grouping (e.g. Group Anagrams)
```java
Map<String, List<String>> groups = new HashMap<>();
for (String word : words) {
    char[] chars = word.toCharArray();
    Arrays.sort(chars);
    String key = new String(chars);
    groups.computeIfAbsent(key, k -> new ArrayList<>()).add(word);
}
```

### d) Detecting Duplicates
```java
Set<Integer> seen = new HashSet<>();
for (int num : nums) {
    if (!seen.add(num)) return true; // add() returns false if already present
}
```

### e) Adjacency List for Graphs
```java
Map<Integer, List<Integer>> graph = new HashMap<>();
graph.computeIfAbsent(u, k -> new ArrayList<>()).add(v);
```

### f) Sliding Window with Character Counts
```java
Map<Character, Integer> window = new HashMap<>();
// expand/shrink window, use window.merge() and window.remove()
```

### g) Caching / Memoization in DP
```java
Map<Integer, Long> memo = new HashMap<>();
long fib(int n) {
    if (n <= 1) return n;
    if (memo.containsKey(n)) return memo.get(n);
    long result = fib(n - 1) + fib(n - 2);
    memo.put(n, result);
    return result;
}
```

## 10. Common Pitfalls & Interview Traps

- **Iterating and modifying** a HashMap at the same time ‚Üí `ConcurrentModificationException`. Use `Iterator.remove()` or `entrySet().removeIf()` instead.
- **Mutable keys**: if a key's `hashCode()` changes after insertion (e.g., mutable object), you can no longer find that entry.
- `get()` on a missing key returns `null`, not an exception ‚Äî always null-check, or use `getOrDefault`.
- Autoboxing traps: `map.get(key) - 1` can throw `NullPointerException` if key is absent (unboxing null Integer).
- `containsValue()` is **O(n)**, unlike `containsKey()` which is O(1).
- HashMap order is **not guaranteed** ‚Äî never rely on it; use `LinkedHashMap` if order matters.
- Worst-case O(n) is possible if many keys collide into the same bucket (mitigated by treeification in Java 8+, but still avoid poor `hashCode()` implementations).

## 11. Quick Comparison Cheat Sheet

| You need... | Use |
|---|---|
| Fast key lookup, no order needed | `HashMap` |
| Insertion order preserved | `LinkedHashMap` |
| Sorted keys / range queries | `TreeMap` |
| Just unique elements, no values | `HashSet` |
| Thread-safe map | `ConcurrentHashMap` |
| LRU Cache | `LinkedHashMap` (with `removeEldestEntry` override) |

---

## 12. Summary ‚Äî What to Remember for Interviews

1. HashMap = array of buckets + hashing + linked list/tree for collisions.
2. Average O(1) for put/get/remove; worst case O(n) or O(log n) post-Java 8.
3. Load factor 0.75, capacity doubles on resize, always a power of 2.
4. `hashCode()` + `equals()` contract is critical for custom keys.
5. Use `merge`, `computeIfAbsent`, `getOrDefault` to write cleaner DSA code.
6. Recognize the **frequency map / two-sum / grouping / adjacency list / memoization** patterns ‚Äî 90% of HashMap-based DSA questions reduce to one of these.
