# Product of Array Except Self — Explained

## Problem

Given an array `nums`, return an array `ans` such that `ans[i]` equals the product of all elements of `nums` except `nums[i]` — without using division and in O(n) time.

Example:
```
nums = [1, 2, 3, 4]
ans  = [24, 12, 8, 6]
```

## Core Idea

For any index `i`:

```
ans[i] = (product of everything to the LEFT of i) × (product of everything to the RIGHT of i)
```

So the problem breaks into two smaller problems:
1. Build a `left` array where `left[i]` = product of all elements **before** index `i`.
2. Build a `right` array where `right[i]` = product of all elements **after** index `i`.
3. Multiply them together, position by position.

## Step-by-Step Walkthrough (`nums = [1, 2, 3, 4]`)

### Step 1: Build `left[]`

`left[i]` = product of everything before index `i`.

```java
left[0] = 1;   // nothing to the left of index 0
for (int i = 1; i < length; i++) {
    left[i] = left[i - 1] * nums[i - 1];
}
```

This works because `left[i-1]` already holds the product of everything before `i-1`. Multiplying it by `nums[i-1]` (the element just added) extends the product up to (but not including) index `i`.

| i | left[i]              | value |
|---|----------------------|-------|
| 0 | (empty)              | 1     |
| 1 | left[0] * nums[0]    | 1 * 1 = 1 |
| 2 | left[1] * nums[1]    | 1 * 2 = 2 |
| 3 | left[2] * nums[2]    | 2 * 3 = 6 |

`left = [1, 1, 2, 6]`

### Step 2: Build `right[]`

`right[i]` = product of everything after index `i`. Same idea, but scanning from the end.

```java
right[length - 1] = 1;   // nothing to the right of the last index
for (int i = length - 2; i >= 0; i--) {
    right[i] = right[i + 1] * nums[i + 1];
}
```

| i | right[i]               | value |
|---|-------------------------|-------|
| 3 | (empty)                | 1     |
| 2 | right[3] * nums[3]     | 1 * 4 = 4 |
| 1 | right[2] * nums[2]     | 4 * 3 = 12 |
| 0 | right[1] * nums[1]     | 12 * 2 = 24 |

`right = [24, 12, 4, 1]`

### Step 3: Combine

```java
for (int i = 0; i < length; i++) {
    ans[i] = left[i] * right[i];
}
```

| i | left[i] | right[i] | ans[i] |
|---|---------|----------|--------|
| 0 | 1       | 24       | 24     |
| 1 | 1       | 12       | 12     |
| 2 | 2       | 4        | 8      |
| 3 | 6       | 1        | 6      |

Final answer: `[24, 12, 8, 6]` ✅

## Why No Division Is Needed

A naive approach would compute the total product and divide by `nums[i]` for each index. This solution avoids division entirely by splitting the "total product except self" into two halves (left side and right side) and combining them — which also sidesteps the edge case of zeros in the array.

## Complexity

- **Time:** O(n) — three separate single passes over the array (build `left`, build `right`, combine).
- **Space:** O(n) for the `left`, `right`, and `ans` arrays.
  - *Optimization:* You can drop `left[]` and `right[]` as separate arrays and instead use the output array `ans` to store the left products first, then do a single backward pass with a running `rightProduct` variable to multiply in the right-side contribution. That reduces auxiliary space to O(1) (excluding the output array).

## Optimized O(1) Extra Space Version

```java
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int length = nums.length;
        int[] ans = new int[length];

        ans[0] = 1;
        for (int i = 1; i < length; i++) {
            ans[i] = ans[i - 1] * nums[i - 1];   // ans[i] holds left product
        }

        int rightProduct = 1;
        for (int i = length - 1; i >= 0; i--) {
            ans[i] = ans[i] * rightProduct;       // multiply in right product
            rightProduct *= nums[i];
        }

        return ans;
    }
}
```

This reuses `ans[]` to store left products first, then walks backward multiplying in the right-side product using a single running variable — same logic, less memory.
