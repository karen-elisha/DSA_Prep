# HashMap in Java — Beginner Notes

## 1. What is a HashMap?

A **HashMap** stores data in **pairs**: a **key** and a **value**. Think of it like a real dictionary — you look up a *word* (key) to get its *meaning* (value).

```java
import java.util.HashMap;

HashMap<String, Integer> map = new HashMap<>();
map.put("apple", 1);
map.put("banana", 2);

System.out.println(map.get("apple")); // 1
```

Here:
- `"apple"`, `"banana"` → **keys**
- `1`, `2` → **values**

**Simple rules to remember:**
- Every key is **unique**. If you `put()` the same key again, it just **overwrites** the old value.
- You can have **one `null` key**, but multiple `null` values.
- The order of items is **not guaranteed** — don't expect them to come out in the order you put them in.

---

## 2. Why use HashMap?

Because it's **super fast** to:
- Add a value
- Find a value
- Check if a key exists

All these operations are (on average) **O(1)** — meaning it takes roughly the same tiny amount of time no matter how big your map is. This is why HashMap is so popular in coding problems.

---

## 3. How does it work? (Simple version)

You don't need deep internals as a beginner, but here's the basic idea:

1. Every key has a **hashcode** (a number generated from the key).
2. Java uses that number to decide which "slot" (bucket) to store the pair in.
3. When you ask for a key later, Java calculates the same number again and jumps straight to that slot — instead of checking every single item one by one.

This is why it's much faster than searching through a list one by one.

---

## 4. Common Methods (the ones you'll actually use)

```java
map.put("apple", 1);         // add or update a key-value pair
map.get("apple");            // get value for a key → returns 1
map.get("mango");            // key doesn't exist → returns null
map.getOrDefault("mango", 0);// returns 0 if key not found (safer!)

map.remove("apple");         // removes the key
map.containsKey("apple");    // true / false
map.containsValue(1);        // true / false

map.size();                  // number of pairs
map.isEmpty();                // true if map has 0 pairs
map.clear();                  // removes everything
```

---

## 5. How to Loop Through a HashMap

```java
HashMap<String, Integer> map = new HashMap<>();
map.put("apple", 1);
map.put("banana", 2);

// Best way — gives you key AND value together
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " -> " + entry.getValue());
}

// Only keys
for (String key : map.keySet()) {
    System.out.println(key);
}

// Only values
for (int value : map.values()) {
    System.out.println(value);
}
```

## 6. The #1 Beginner Use Case: Counting Things

This is the pattern you will use **all the time** in DSA problems — counting how many times something appears.

```java
String s = "banana";
HashMap<Character, Integer> freq = new HashMap<>();

for (char c : s.toCharArray()) {
    if (freq.containsKey(c)) {
        freq.put(c, freq.get(c) + 1);   // already seen → increase count
    } else {
        freq.put(c, 1);                  // first time seeing it → count = 1
    }
}

System.out.println(freq); // {a=3, b=1, n=2}
```

**Shortcut version** (once you're comfortable):
```java
freq.put(c, freq.getOrDefault(c, 0) + 1);
```

## 7. Beginner-Friendly Example: Two Sum Problem

A classic first problem that shows why HashMap is powerful.

**Problem**: Given an array of numbers and a target, find two numbers that add up to the target.

```java
public int[] twoSum(int[] nums, int target) {
    HashMap<Integer, Integer> map = new HashMap<>(); // value -> index

    for (int i = 0; i < nums.length; i++) {
        int needed = target - nums[i];

        if (map.containsKey(needed)) {
            return new int[]{map.get(needed), i}; // found the pair!
        }

        map.put(nums[i], i); // remember this number and its position
    }

    return new int[]{-1, -1}; // no answer found
}
```

**Why this is fast**: instead of checking every pair of numbers (slow, O(n²)), we remember numbers we've already seen, so we can check "have I seen the number I need?" instantly.

## 8. Checking for Duplicates

```java
HashMap<Integer, Boolean> seen = new HashMap<>();
int[] nums = {1, 2, 3, 2};
boolean hasDuplicate = false;

for (int num : nums) {
    if (seen.containsKey(num)) {
        hasDuplicate = true;
        break;
    }
    seen.put(num, true);
}
```

(Note: for just checking duplicates, `HashSet` is actually the better tool — see below.)

## 9. HashMap vs HashSet (Quick Note)

- **HashMap** → stores key-value pairs. Use when you need to link two things together (e.g., word → count).
- **HashSet** → stores only unique values, no pairs. Use when you just need to check "have I seen this before?"

```java
HashSet<Integer> set = new HashSet<>();
set.add(5);
set.contains(5); // true
```

## 10. Common Beginner Mistakes

1. **Forgetting `get()` can return `null`**
   ```java
   Integer count = map.get("mango"); // null if key doesn't exist
   count + 1; // ❌ crashes with NullPointerException!
   ```
   Fix: use `getOrDefault("mango", 0)` instead.

2. **Expecting order** — HashMap does NOT keep insertion order. If you need order, use `LinkedHashMap`.

3. **Using `==` to compare values** — for objects like `Integer` or `String`, use `.equals()`, not `==`.

4. **Modifying the map while looping over it** — this causes an error. If you need to remove things while looping, use an `Iterator`.

---

## 11. Cheat Sheet Summary

| What you want to do | Method |
|---|---|
| Add/update a pair | `map.put(key, value)` |
| Get a value | `map.get(key)` |
| Get safely with default | `map.getOrDefault(key, default)` |
| Remove a pair | `map.remove(key)` |
| Check if key exists | `map.containsKey(key)` |
| Check if value exists | `map.containsValue(value)` |
| Loop through pairs | `map.entrySet()` |
| Count of pairs | `map.size()` |
